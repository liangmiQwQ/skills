# Design Common Sayings — 常见说辞

> **Load when:** Feeling the urge to shortcut a design step, or the user pushes for speed over quality.
> **Why it matters:** Design rationalizations usually sound reasonable in the moment. This table preemptively debunks them.

---

| Saying / Rationalization | Reality |
|---|---|
| "The user won't notice this alignment" | Alignment errors accumulate. Users notice the gestalt, not individual pixels, but the gestalt depends on every pixel being intentional. |
| "We'll fix spacing in the final polish" | Spacing is structural, not cosmetic. Decide the spacing scale before the first wireframe, not after. Resizing every margin later costs more than getting it right the first time. |
| "More content on one page = more value" | Information density without hierarchy = no information. Readers need breathing room and a clear focal point. |
| "Just make it work, design doesn't matter" | For users, visual quality IS a component of trust. A functional but visually broken product is perceived as unreliable. |
| "This gradient looks cool" | Most gradients are decoration, not communication. If it does not serve hierarchy or brand identity, it is noise. |
| "I will find the brand colours later" | Guessing brand colours = guaranteed rework. Extract from real assets now, or accept the risk of a full rebuild. |
| "It is just a prototype" | Prototypes set expectations. A sloppy prototype signals a sloppy product. The same quality standards apply — the difference is scope, not rigour. |
| "This looks good in my editor" | The browser is the truth. Open it, screenshot it, verify it. Editors lie about fonts, spacing, colour, and rendering. |
| "I will add responsive support later" | Responsive is structural, not cosmetic. Decide the layout system now or rebuild every section later. |
| "This font is close enough" | Font selection is brand identity. The wrong font = the wrong brand voice. Verify the exact family and weight. |
| "The first screen looks good, the rest is probably fine" | The first screen gets the most attention. Every section you touched must be inspected independently. Visited sections that are untouched are fine; touched sections are not. |
| "This is a common pattern, it will work" | Common patterns work in common contexts. Your content may not be common. Verify the pattern with your actual data, not generic expectations. |
| "I can optimise the colours later" | Colour is the first thing users perceive. Wrong colour = wrong first impression. Choose the palette before building, not after. |
| "The design system is flexible enough" | Design systems constrain by design. If a page breaks the system, either the page needs a different approach or the system needs an extension. Do neither silently. |
| "Reuse the same layout for every section" | Repetition is a pattern; sameness is monotony. Vary density and composition by content type. Keep the grid, change the rhythm. |

## When to read

Read this table BEFORE opening any conversation with "The user won't notice..." or "I'll fix it later..."
